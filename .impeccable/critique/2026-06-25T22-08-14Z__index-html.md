## Anti-Patterns Verdict

**LLM assessment**: The Intel entry does not read as AI-generated in the slop sense. The business-case framing ("which region should Intel site its next data center?") is concrete and specific, not generic. The recommendation-first structure differentiates this entry from the adjacent "here is a list of charts I made" project. Copy is tight and avoids marketing voice. No side-stripe accents, no gradient text, no ghost-card double-decoration, no eyebrow kickers. The expand/collapse pattern is borrowed exactly from the existing entries — no new components invented. 2. **Correct animation wiring.** `reveal-item` class, `data-delay="240"` (staggering correctly after Job Salary's 160), and the expand/collapse ARIA pattern (`aria-expanded`, `aria-controls`, `aria-hidden`) all match the existing entries exactly. This is the area most likely to introduce bugs when cloning, and it's clean.

## Priority Issues

**[P2] Section label divergence: "Dashboard Screenshots" vs "Screenshots"**
- **Why it matters**: Both NYC Jobs and Job Salary use "Screenshots" as the section label. "Dashboard Screenshots" reads as slightly redundant (all screenshots in this section are dashboard screenshots) and breaks the visual rhythm a returning recruiter has learned from the earlier cards.
- **Fix**: Change `Dashboard Screenshots` → `Screenshots` to match the existing pattern.
- **Suggested command**: `$impeccable polish`

**[P2] `<strong>` inside bullet text (not used elsewhere)**
- **Why it matters**: No other project card uses inline bold in the `project-bullets` list. Two `<strong>` instances — one in the visible bullet, one inside the detail — are the only inline formatting on the entire page. They read as emphasis spray rather than deliberate system behavior.
- **Fix**: Either (a) remove the `<strong>` from the visible bullet — the recommendation is already the leading clause, so it reads clearly without it — or (b) adopt this as a deliberate system pattern and add the same treatment to the CENT finding inside the detail only (where the conclusion needs punctuation). Option (a) is the path of least resistance; option (b) requires touching the other entries to feel intentional.
- **Suggested command**: `$impeccable polish`

**[P3] No footer action at card bottom**
- **Why it matters**: NYC Jobs and Job Salary both end with a `project-repo-link` anchor positioned after the `project-detail` div. The Intel card ends with the expand button (when collapsed) or the image grid (when expanded). There is no GitHub repo for this project, so the absence is correct — but the visual weight of the card bottom differs slightly. A Tableau Public link, a note about the `.twbx` file, or a simple "No public repo" repo-note label would close the card consistently.
- **Fix**: Add a `<span class="project-repo-note">Tableau workbook — not publicly hosted</span>` where the repo link would sit, or omit entirely and accept the divergence.
- **Suggested command**: `$impeccable polish`

**[P3] 8-image grid density in expanded state**
- **Why it matters**: The existing entries have 1 (NYC Jobs) and 2 (Job Salary) screenshots. 8 images renders as a full 3-column grid on desktop (3 + 3 + 2) — significantly more visually dense than anything else on the page when expanded. It is not broken, but a recruiter opening the detail to "take a quick look" may feel the wall effect.
- **Fix**: Keep all 8 images but consider promoting `Final_Intel_Dashboard.png` as a featured image above the fold of the detail (larger, full-width) and grouping the 7 individual views in the grid below it. This creates a "hero shot → supporting detail" hierarchy that matches how a Tableau workbook is actually presented.
- **Suggested command**: `$impeccable layout`
