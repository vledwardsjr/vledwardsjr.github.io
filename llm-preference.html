Research Question

Can a model trained on measurable textual features predict which of two AI responses a human will prefer? And do specific signals — structural flow and response specificity — reliably drive that preference, or is human judgment too subjective to learn from?

Motivation

This project is directly motivated by professional experience as an AI output evaluator. In close preference decisions between two strong responses, the deciding factor consistently came down to two signals: how well the response was structured, and how specific (vs. vague) its content was. This project formalizes and quantifies those intuitions — testing whether a machine can learn what an experienced human evaluator already knows.

Dataset

Chatbot Arena Human Preference Dataset by LMSYS, sourced from Hugging Face. Contains real-world human votes on which of two AI-generated responses was preferred. Each record includes:

  • A user prompt
  • Response A and Response B from two different LLMs
  • A human winner vote (A wins, B wins, or tie)

Connection to RLHF

This project mirrors the structure of real-world Reinforcement Learning from Human Feedback (RLHF) pipelines: collect human preference data, extract signals, and train a reward model. The pairwise classification architecture — featurizing both responses and comparing them before classification — is a direct analogue of how preference reward models are built in production AI systems.

Project Roadmap

Phase 1 Data Acquisition & EDA Current Phase
  • Load and inspect the Chatbot Arena dataset from Hugging Face
  • Explore vote distribution, response length patterns, and label balance
  • Surface potential confounds — e.g., does response length alone predict the winner regardless of quality?
Phase 2 Feature Engineering Upcoming
  • Translate qualitative signals (structural flow, specificity / vagueness) into computable numeric features
  • Engineer pairwise features: difference scores between Response A and B on each signal
  • Validate that engineered features are independent of raw length (to avoid confounding with the length bias surfaced in EDA)
Key challenge: converting fuzzy human judgment into measurable, computable input a model can learn from — without losing the signal that makes those features meaningful.
Phase 3 Modeling Upcoming
  • Logistic Regression baseline — establishes interpretability floor and a reference accuracy to beat
  • Random Forest and XGBoost for improved predictive performance
  • Pairwise input architecture: both responses featurized and compared before binary classification
Phase 4 Evaluation & Error Analysis Upcoming
  • Metrics: accuracy, precision, recall, F1, ROC-AUC
  • Feature importance analysis — which signals actually drove model predictions, and how closely do they match the original hypothesis?
  • Error analysis: where and why the model disagrees with human raters, and what that reveals about the limits of automating evaluation
Phase 5 Delivery Upcoming
  • Streamlit interactive demo — paste two AI responses, receive a prediction and explanation of which wins and why
  • Written findings and methodology report covering the full pipeline and key results
  • Polished GitHub repository with complete documentation
